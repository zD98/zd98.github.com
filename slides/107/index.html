JS拖拽总结

被面试问到了拖拽原理... 我光荣的忘了... 特此写下总结

  1. 拖拽初级

    - 事件mousedown, mousemove, moveup
    - 监听拖拽元素的mousedown事件
    - 在mousedown事件下 添加mousemove, mouveup 的事件处理
    - 技能：get 事件的处理 拖拽原理
    - 缺点:每个元素都需要添加事件, 比较繁琐, 不易拓展, 不易团队使用

  2. 拖拽进阶 第一版(来自于JS高级程序设计)

    - 添加了事件委托, 将mousedown, mouseup, moumove 等事件添加到了document上
    - 原理：通过event.target获取触发事件的元素, 进行判断触发mousedown是否是可拖拽元素
    - mousemove, mouseup 判断是否有dragging
    - 事件添加到document or body 是因为mousemove的时候会出现bug
    - 写了container与item的类 所有的container共享一个dragging(活动拖拽),
    然后根据left,top,等判断进入哪个容器
    - get JS事件 事件委托 闭包 类
    - 缺点:虽然简化了事件的部分, 拖拽没有container的限制 会随意乱拽 代码耦合太深 无法
    相互关联太多
    依然没有什么可用性, 更像是知识点学习...

  3. 拖拽进阶 第二版 (来自于jQuery源码)

    - 添加了manager的类 将container与item 解耦和
    - 在manager中负责添加与控制拖拽事件,以及发生拖拽事件时通知响应container与item
    - container负责容器的处理,属性储存, item 未知
    - get：设计模式
    - 缺点:没有彻底的解耦和 关于item的部分 并没有做什么 将dom的控制都扔给了manager
    manager 和 item没有解耦

  4. 拖拽进阶 未完成版 

    - 将css 的变化 添加到类中 好吧 是我之前太懒了
    - 解耦 container 负责实现添加item 移入移出等 样式变化
    item负责 实现 拖拽时的样式变化 等
    - manager 只是负责拖拽的逻辑处理
    - 消息传递的话 应该用topic(观察者模式)吧 jquery 用的是互相的函数调用 但我觉得这样函数调用不ok啊
    - 写一个helper 就是拖拽移动的那个元素
    - 看看能不能在angular上再写一个 唔先这样
